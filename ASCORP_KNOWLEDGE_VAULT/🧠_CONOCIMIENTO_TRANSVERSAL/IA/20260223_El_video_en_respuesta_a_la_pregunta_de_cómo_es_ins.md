*   **Stack Tecnológico:**
    *   **Meta Pixel:** Fragmento de código JavaScript para seguimiento de eventos en sitios web.
    *   **Lovable AI:** Plataforma de inteligencia artificial que, por inferencia, ofrece funcionalidades de marketing, e-commerce, o gestión de usuarios, requiriendo seguimiento de eventos.
    *   **Meta Business Suite/Ads Manager/Events Manager:** Interfaz de administración de Meta para configuración de negocios, anuncios y seguimiento.
*   **Algoritmos/Procesos:**
    1.  **Generación/Recuperación del Pixel:** En Meta Events Manager, se genera un ID único para el Pixel y se proporciona el código base (JavaScript `fbq()` y `noscript` fallback).
    2.  **Inyección de Código:** El código del Pixel es copiado y luego pegado en una sección específica de la configuración de Lovable AI (típicamente en el `<head>` del HTML), asegurando que se cargue en todas las páginas donde se necesite seguimiento.
    3.  **Activación de Eventos:** Al cargar una página donde el Pixel está instalado, se dispara un evento `PageView` por defecto. Eventos adicionales (Lead, Purchase, AddToCart, etc.) se pueden configurar mediante la API de eventos del Pixel o el Event Setup Tool.
    4.  **Transmisión de Datos:** Los eventos capturados por el Pixel se envían a los servidores de Meta, donde se procesan para atribución, creación de audiencias, optimización de campañas y reporting.
    5.  **Verificación:** La herramienta "Probar Eventos" en Meta Events Manager permite simular la navegación del usuario para confirmar que el Pixel está recibiendo correctamente los eventos.

## 📝 BITÁCORA DE DETALLES "INVISIBLE"
*   **Colocación Estratégica:** El Pixel DEBE colocarse lo más arriba posible dentro de la etiqueta `<head>` del sitio web para asegurar que se cargue antes que cualquier otro script y capture la mayor cantidad de eventos posible, incluso si el usuario abandona rápidamente.
*   **Eventos Estándar vs. Personalizados:** Además del `PageView`, es crucial configurar eventos estándar de Meta (e.g., `Lead`, `Purchase`, `AddToCart`) relevantes para la funcionalidad de Lovable AI, y opcionalmente, eventos personalizados con parámetros específicos para un análisis más profundo.
*   **API de Conversiones (CAPI):** Para una resiliencia mayor frente a los bloqueadores de cookies y cambios en la privacidad, se debería considerar complementar el Pixel con la API de Conversiones de Meta, enviando eventos directamente desde el servidor de Lovable AI. Esto mejora la precisión y la entrega de datos.
*   **Consentimiento de Cookies (GDPR/CCPA):** Es imperativo integrar la instalación del Pixel con una solución de gestión de consentimiento (CMP) para cumplir con las regulaciones de privacidad. El Pixel solo debe activarse después de que el usuario haya dado su consentimiento explícito para el seguimiento.
*   **Depuración del Pixel:** Utilizar la extensión "Meta Pixel Helper" para navegadores Chrome es una herramienta indispensable para depurar la instalación y verificar qué eventos se están disparando en cada página.
*   **Múltiples Píxeles:** Si se manejan diferentes propiedades o clientes dentro de Lovable AI, se debe asegurar que cada uno tenga su Pixel correspondiente instalado y configurado para evitar la mezcla de datos.
